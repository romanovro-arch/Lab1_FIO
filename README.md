# Лабораторная работа №1. [cite_start]Проект 1: Вычисление вида треугольника [cite: 1, 10]

**Выполнил(а):** Романов Р.О.

## Описание проекта
[cite_start]Данный проект представляет собой десктопное приложение (WPF / C#), которое принимает на вход длины трех сторон, вычисляет вид треугольника и определяет координаты трех его вершин для последующей отрисовки в поле 100х100 px[cite: 11, 49].

## Входные данные
* [cite_start]**Сторона А** (строка) [cite: 13]
* [cite_start]**Сторона Б** (строка) [cite: 14]
* [cite_start]**Сторона В** (строка) [cite: 15]

[cite_start]**Ограничения:** Все три строки должны представлять вещественные положительные числа[cite: 16]. [cite_start]Ошибочные данные (текст, отрицательные числа) обрабатываются корректно и не приводят к прерыванию работы программы (падению приложения)[cite: 17].

## Выходные данные
1. [cite_start]**Тип треугольника:** «равносторонний», «равнобедренный», «разносторонний», «не треугольник» или пустая строка (для нечисловых данных)[cite: 19].
2. [cite_start]**Координаты:** Список из трех точек `(X, Y)`, отмасштабированных для отображения в области 100x100 px[cite: 20].
   * [cite_start]Если введенные числа не могут образовать треугольник, координаты принимают значение `(-1, -1)`[cite: 21].
   * [cite_start]Если введены невалидные (нечисловые) данные, координаты принимают значение `(-2, -2)`[cite: 22].

## Логирование
[cite_start]В проекте настроено логирование с использованием библиотеки **Serilog**[cite: 51]. [cite_start]Логи дублируются в консоль отладки и в текстовый файл (по умолчанию сохраняется в папку `logs/file_.txt` рядом с исполняемым файлом)[cite: 25, 66].

* [cite_start]**Успешные запросы** (когда удалось построить реальный треугольник) сохраняют дату-время, входные параметры и результаты вычислений[cite: 23, 26].
* [cite_start]**Неуспешные запросы** (некорректный ввод, невозможный треугольник или системная ошибка) фиксируют причину отказа и, при необходимости, трассировку стека исключения[cite: 24, 27].

## Инструкция по запуску
1. Склонируйте репозиторий на локальную машину.
2. [cite_start]Откройте проект в Visual Studio[cite: 49].
3. [cite_start]Дождитесь автоматического восстановления NuGet-пакетов (`Serilog`, `Serilog.Sinks.File`, `Serilog.Sinks.Console`)[cite: 59].
4. Соберите и запустите проект.
